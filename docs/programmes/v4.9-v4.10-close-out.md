## Completed gates

1. `scripts/dumpArchitecture.mjs` is repository-owned, self-including and
   verifies all docs, workflows, Wrangler configs and declaration files.
2. The TypeScript-AST `AnyKeyword` baseline has per-file and total ceilings;
   engines are locked at zero.
3. `src/shared/cachePolicy.ts` registers each cache by storage layer, expiry
   strategy, backup classification and invalidation owner. IndexedDB cache
   completeness is reconciled against the backup registry.
4. The full ZIP test exports a registry-complete real IndexedDB fixture, deletes
   the database, opens a fresh database and imports it. Separate tests retain
   damaged-source rejection and prove missing archive media leaves the target
   untouched.
5. `fast-check` runs bounded independent-model lifecycle sequences and bounded
   arbitrary backup inputs against the existing throwing validation contract.
6. `FieldGuideController.tsx` is 15 lines. Interaction composition moved to the
   FieldGuide workspace, prediction/question persistence moved to a dedicated
   post-scan orchestrator, and all page `db` value imports are gone. Type-only
   record imports remain valid.
7. BGS and Wales LiDAR behavior was characterized before change. All four
   Workers use compatibility date `2026-07-23`; static and geocode bindings are
   generated by Wrangler and checked in the Worker CI command.
